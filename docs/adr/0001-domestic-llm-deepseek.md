# ADR 0001: 采用国产大模型，主力使用 DeepSeek

## 状态

已接受 (2026-06-16)

## 背景

中国大陆无法直接访问 OpenAI GPT-4、Claude 等海外大模型服务。项目需要选择一个既能满足代码生成质量要求，又符合合规约束的模型方案。

可选路径：

1. **海外模型 + 代理**：通过香港 Azure 或 API 代理访问 GPT-4/Claude，代码生成质量最高，但处于合规灰色地带，随时可能被封堵。
2. **国产大模型**：DeepSeek V3/Coder、通义千问 Qwen Coder、智谱 GLM 等均可在中国大陆直接调用，完全合规。
3. **混合策略**：主力国产 + 复杂场景 fallback 海外。

## 决策

**采用纯国产大模型方案，主力模型为 DeepSeek V3/Coder。**

- 所有代码生成调用 DeepSeek API（api.deepseek.com，中国大陆可直接访问）。
- 不看海外模型，不走代理，不接受合规风险。
- 如后续 DeepSeek 在特定场景下质量不足，可评估引入通义千问作为补充。

## 理由

1. **合规无风险**：不需要任何跨境数据传输或 API 代理，满足中国大陆监管要求。
2. **代码能力足够**：DeepSeek V3 和 DeepSeek Coder 在代码生成评测中处于国产模型领先水平，对 React/Vue/HTML 等前端技术栈有较好的理解。
3. **成本可控**：DeepSeek API 定价低于 GPT-4，适合 Solo 项目起步阶段的成本模型。
4. **可自部署**：DeepSeek 开源权重，必要时可自建推理服务，不依赖单一 API 供应商。

## 后果

- **正向**：合规零风险，API 稳定可预期，成本低于海外方案。
- **负向**：代码生成质量可能在某些复杂场景下低于 GPT-4/Claude，需要更精心的 prompt 工程来弥补。
- **需关注**：DeepSeek 作为创业公司的 API 稳定性，建议预留模型切换能力（通过 adapter 层抽象模型调用）。
